# Docker Lab: Контейнеризація аналітичного пайплайну
Цей проєкт демонструє мікросервісну архітектуру для автоматизованої обробки, аналізу та візуалізації даних з використанням Docker та Docker Compose.

## Архітектура проєкту
Проєкт складається з 6 незалежних контейнерів (сервісів), які взаємодіють між собою через внутрішню Docker-мережу та спільні томи (volumes):

db (PostgreSQL) — база даних для зберігання початкового набору даних.

data_load — зчитує вхідний файл dataset.csv та імпортує його в таблицю бази даних.

data_quality_analysis — підключається до БД, перевіряє дані на якість (пропуски, дублікати) та генерує звіт у форматі JSON.

data_research — виконує обчислення базових статистик та групування даних, формуючи JSON-звіт.

visualization — будує аналітичні графіки (розподіл за статтю, кількість за званнями) та зберігає їх у форматі PNG.

web (Flask) — веб-сервер, який збирає всі згенеровані звіти та графіки і відображає їх на єдиній HTML-сторінці.

## Інструкція із запуску
### Попередні вимоги
Встановлений та запущений Docker (Docker Desktop для Windows/Mac або Docker Engine для Linux).

Вхідний файл даних повинен знаходитися за шляхом: data/raw/dataset.csv.

### Кроки запуску
1. Відкрийте термінал у кореневій папці проєкту (там, де знаходиться файл compose.yaml).

2. Виконайте команду для збірки образів та запуску всіх контейнерів:

docker compose up --build
3. Процес запуску автоматизовано. Сервіси запускатимуться послідовно (завдяки depends_on). Дочекайтеся, поки в терміналі з'явиться повідомлення про успішний запуск веб-сервера:



lab_web  |  * Running on http://172.18.0.x:5000
## Перегляд результатів
Після успішного запуску всіх сервісів, веб-інтерфейс з результатами аналізу та візуалізаціями буде доступний у вашому браузері за адресою:
http://localhost:5000

## Зупинка проєкту
Щоб коректно зупинити роботу всіх сервісів:

Натисніть Ctrl + C у терміналі, де працює Docker Compose.

Для повного очищення зупинених контейнерів виконайте команду:


docker compose down
(Примітка: Дані бази даних зберігаються локально у томі pgdata, тому вони не будуть втрачені при перезапуску).
